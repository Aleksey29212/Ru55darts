# DartBrig Pro - Система управления рейтингами (v2.8 Stable)

Современная платформа для автоматизации дартс-турниров и управления профессиональными профилями игроков.

## 🚀 Готовность к деплою (Any Hosting Ready)
Система полностью автономна и может быть развернута на любом хостинге:
- **Vercel / Netlify**: Рекомендуется для Next.js (авто-сборка из GitHub).
- **Firebase App Hosting**: Прямая интеграция с Google Cloud.
- **Docker / VPS**: Поддержка Standalone-режима сборки.

## ☁️ Подключение Облачной Базы
Облачная база данных (Firebase) подключается к любому хостингу через переменные окружения. Просто добавьте следующие ключи в панель управления вашего хостинга (Environment Variables):

```env
NEXT_PUBLIC_FIREBASE_API_KEY=ваш_ключ
NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN=ваш_домен_авторизации
NEXT_PUBLIC_FIREBASE_PROJECT_ID=ваш_id_проекта
NEXT_PUBLIC_FIREBASE_APP_ID=ваш_id_приложения
NEXT_PUBLIC_ADMIN_PASSWORD=ваш_пароль_входа_в_админку
```

*Если эти переменные не заданы, система автоматически запустится в **Demo-режиме** (данные будут храниться в оперативной памяти сервера до перезагрузки).*

---

## 📦 Основные компоненты
- **Frontend**: Next.js 15 (App Router) + Tailwind CSS.
- **UI**: 10 Мастер-шаблонов личных карточек с защитой от искажения цифр.
- **Backend**: Firebase Firestore (универсальное хранилище турниров и профилей).
- **AI**: Google Genkit (интеллектуальная генерация тем оформления).

## 📸 Хранение фотографий
Все фотографии, загруженные через **«Фотостудию»**, конвертируются в формат **Base64** и сохраняются внутри документа игрока в базе данных. Это гарантирует:
1. **Постоянство**: Фото никогда не потеряются при смене хостинга.
2. **Скорость**: Мгновенная загрузка без обращения к внешним S3-хранилищам.
3. **Автономия**: Данные и медиа всегда находятся «внутри» вашего проекта.

## 🛠️ Команды управления

1. **Загрузка в GitHub**:
   ```bash
   npm run deploy:github
   ```
2. **Локальная проверка сборки**:
   ```bash
   npm run build
   ```

Разработано специально для DartBrig. Все данные (архивы турниров, профили, фото) сохраняются в облачном «контейнере» и не требуют повторного парсинга.
