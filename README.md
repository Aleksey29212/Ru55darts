# DartBrig Pro - Система управления рейтингами (v2.8 Stable)

Современная платформа для автоматизации дартс-турниров и управления профессиональными профилями игроков. Построена на стеке **Next.js 15**, **Tailwind CSS** и **Firebase**.

## 🛠 Технический стек
- **Framework**: Next.js 15 (App Router)
- **Runtime**: Node.js 20+
- **Backend**: Firebase Firestore (NoSQL)
- **Auth**: Firebase Auth (Anonymous)
- **Output**: `standalone` (оптимизировано для Docker и Node-хостингов)

## 🚀 Инструкция по деплою (Timeweb Cloud / Vercel)

### Шаг 1: Отправка в GitHub
1. Откройте терминал.
2. Выполните команду: `npm run deploy:github`
3. Следуйте подсказкам скрипта для подключения вашего репозитория.

### Шаг 2: Настройка на хостинге (Timeweb)
1. Создайте проект «Node.js приложение» в Timeweb Cloud.
2. Выберите ваш GitHub репозиторий.
3. В разделе **«Переменные окружения»** добавьте ключи из вашего проекта Firebase:
   ```env
   NEXT_PUBLIC_FIREBASE_API_KEY=ваш_ключ
   NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN=ваш_домен
   NEXT_PUBLIC_FIREBASE_PROJECT_ID=ваш_id_проекта
   NEXT_PUBLIC_FIREBASE_APP_ID=ваш_id_приложения
   NEXT_PUBLIC_ADMIN_PASSWORD=ваш_пароль_админа (например, 1234)
   ```
4. Настройки сборки:
   - Команда сборки: `npm run build`
   - Команда запуска: `npm start`
   - Порт: `3000` (или `9002`)

## 📸 Хранение данных
- **Турниры**: Сохраняются в Firestore. Повторный парсинг не требуется.
- **Фотографии**: Загружаются через «Фотостудию», конвертируются в **Base64** и хранятся прямо в документе игрока в Firestore. Это обеспечивает полную автономность — внешние хранилища (S3) не нужны.
