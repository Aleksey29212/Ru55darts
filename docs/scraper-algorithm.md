# Алгоритм парсинга турниров DartBrig Pro (Спецификация для ИИ)

Данный документ описывает эталонную логику работы модуля `src/lib/scraping.ts`. Следуйте этим шагам для воспроизведения или расширения функционала на другие сайты.

## 1. Инициализация (Initialization)
- **Вход**: Строка `input` (ID или URL), `leagueId`, `scoringSettings`.
- **Процесс**: 
    - Если `input` не начинается с `http`, формируется URL для dartsbase.ru: `https://dartsbase.ru/tournaments/{id}/stats`.
    - Выполняется HTTP-запрос с `User-Agent` настольного браузера для обхода базовых проверок на ботов.

## 2. Извлечение метаданных (Metadata Extraction)
- **Название турнира**: Поиск первого тега `h1`. Очистка от вложенных `span` и тегов для получения чистого текста.
- **Дата проведения (Критический шаг)**:
    - Поиск паттерна даты `(\d{2})\.(\d{2})\.(\d{4})` (ДД.ММ.ГГГГ) внутри заголовка `h1`.
    - Если дата найдена, она преобразуется в ISO строку. 
    - **Важность**: Реальная дата из названия имеет приоритет над датой импорта, чтобы графики прогресса в профилях игроков отображались корректно.

## 3. Динамическое сопоставление колонок (Column Mapping)
*Никогда не используйте фиксированные индексы столбцов!*
- **Алгоритм**:
    1. Найти заголовок таблицы (`thead tr`).
    2. Пройтись по всем ячейкам `th`.
    3. Привести текст каждой ячейки к нижнему регистру и искать ключевые слова:
        - `avg` -> Средний набор
        - `180` -> Максимумы
        - `hi-out`, `checkout` -> Максимальное закрытие
        - `rank`, `#`, `место` -> Итоговая позиция
        - `игрок`, `player`, `имя` -> Данные игрока
    4. Сохранить найденные индексы в карту (Map).

## 4. Обработка данных игроков (Data Normalization)
- **ID игрока**: Извлекается из последнего сегмента пути ссылки (`href`) в ячейке с именем. Если ссылки нет — формируется из имени (slug).
- **Числа**: Замена запятых на точки в AVG перед преобразованием в `float`.
- **Аватар**: Автоматическая генерация `picsum.photos/seed` на основе имени для единообразия.

## 5. Расчет очков (Scoring Integration)
- После формирования "сырого" объекта `TournamentPlayerResult`, вызывается функция `calculatePlayerPoints`.
- Она применяет правила выбранной лиги (баллы за место + бонусы за статистику) и формирует финальное значение `points`.

## 6. Контракт данных (Output Contract)
Результатом работы должен быть объект `ScrapedTournament`:
```typescript
{
  id: string;        // ID источника
  name: string;      // Название без лишних тегов
  date: string;      // ISO дата (извлеченная из текста или текущая)
  players: TournamentPlayerResult[]; // Список участников с рассчитанными очками
}
```
